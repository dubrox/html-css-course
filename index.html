Preprocessors

Writing "vanilla" CSS can be verbose and hard to maintain, so there are some tools to express style definitions in more coincise ways and then generate the corresponding CSS files.

This page uses Sass, with SCSS files.

More options exist, look for your favourite by searching for "CSS preprocessor" on the Internet.

Introducing preprocessors to a project requires some extra set-up steps and adds processing time on the development device.
On the other hand, reduces the amount of code to write manually and also automates some tasks, like minification and the inclusion of backward compatibility code.

Selectors

CSS selectors are patterns used to define what part of the web page to apply the styling.

The "cascading" caracteristic of CSS defines also a priority system to determine what styling to apply in case of conflicting definitions. These are based on the code position of the declaration and its specificity.

The most specific definition takes priority. In case of equal specificity, definitions that appear later in the code take priority.

An exeption to the specificity rule applies when CSS layers are used.

Positioning: floating and negative margins

Floating elements can get text flowing around them, useful for images or other figures. The way the text flows around the floating elements is considering the element as a rectangular shape. This shape can be changed through the shape-outside property.

Negative margins are sometimes used to change the position of an element relative to its initial position, moving also its occupied shape. Note that using a position: relative would lead to a different outcome, in which the initial occupied shape is unaffected.

In the past, before flexbox and grid layouts were available in CSS, float positioning was heavily used to define web pages layouts.
As an example, you can inspect the code in Amazon.com back in 2011.

Before float positioning was available, web layouts were built using HTML tables (again, you can inspect this snapshot of Amazon.com back in 2006).
Grid and Flexbox positioning methods came to make front end developers lifes easier and more enjoyable :)

Animation: transformations, transitions and keyframes

Transformation functions allow to translate, rotate, skew and apply those movements even in 3D contexts.

For simple animations from an initial state to an altered one, the transition property can be used. For more complex scenarios, the animation property with the @keyframes block will allow to customize the type of movements and timing.

For some extreme examples of 3D animations with CSS only, visit Amit Sheen's creations.

Responsive design

Device's screen sizes vary broadly and there is no standard set of aspect ratios, which makes it impossible to use that same layout for everyone.

Responsive design is a strategy to ensure that most users will have a good experience browsing a web page, by defining flexible layouts that can adapt to minor differences in screen sizes (i.e. 16:9 vs 4:3), and conditional layouts definitions for mayor differences (i.e. portrait vs landscape viewport).

The conditional definition of CSS rules based on device screen features is obtained through the use of media queries, either in HTML <link media="[query]" or in the CSS code itself, through @media [query] { ... }.

Performance optimization

Page loading time and navigation smoothness are important for an enjoyable user experience, and can cause loss of user base if neglected.

The optimization of loading times at the HTML and CSS level is mainly dealth by reducing the amount of data required to render the page and by splitting the resources inbetween files to allow concurrent downloads.
The Critical Rendering Path (CRP) refers to the minimal resources required to properly render the web page, with special attention to the part of the page initially visible to the user (above the fold).

As an example of missed optimization is the photo to the right: even if it uses the lazy loading technique, it is high resolution image displayed at a fraction of its original size.
This means that even if the user gets to see a low-res version of it, the browser had to load a file way bigger than needed, slowing down the overall experience.
